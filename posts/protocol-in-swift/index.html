Protocol in Swift

What is Protocol?

According to Apple Documentation–

A protocol defines a blueprint of methods, properties, and other requirements that suit a particular task or piece of functionality. The protocol can then be adopted by a class, structure, or enumeration to provide an actual implementation of those requirements. Any type that satisfies the requirements of a protocol is said to conform to that protocol. In addition to specifying requirements that conforming types must implement, you can extend a protocol to implement some of these requirements or to implement additional functionality that conforming types can take advantage of.

Protocols are effectively contracts in Swift: they define a set of methods and properties that a type must implement if it says it conforms to the protocol. If in the real world contract means an agreement of more than one parties it can be an organization, people, etc.

If you agree with the contract you should implement whatever in the contract. It also happens on Protocol, protocol is a contract. If any class want to use that protocol it should be following by implement all the function, properties inside the class.

Example – Protocol:

protocol Employee {
  var name: String { get set }
  var status: String { get set }
  func doWork()
}

and this how you can confirm that protocol in Company class.

class Executive: Employee {
   var name: String
   var status: String

   init(name: String, status: String) {
       self.name = name
       self.status = status
   }

   func doWork() {
       print("Hello this is do Work")
   }
}

But using Extension you also can provide the default implementation for the function.

extension Employee {
    func doWork() {
        print("Hey this is the default implementation")
    }
}

So, let’s say in Company class you don’t want to confirm the protocol doWork function. You can provide the default implementation using Extension.

Here is the example of implementation:

let executive = Executive(name: "test", status: "hello")
executive.doWork()

Reference: https://docs.swift.org/swift-book/LanguageGuide/Protocols.html
